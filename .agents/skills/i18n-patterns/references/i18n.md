# Internationalization Reference

Use this reference when the repository includes or is adopting an i18n framework.

## Project compatibility baseline (mandatory)

Use these conventions when wiring i18n runtime:

- Hybrid key style:
  - plain-English keys for common copy,
  - dotted domain keys for structured feature copy,
  - plural suffix keys (`*_one`, `*_other`).
- `keySeparator: false`
- `nsSeparator: false`
- `fallbackLng: "en"`
- `returnNull: false`
- `returnEmptyString: false`
- `interpolation.escapeValue: false`
- `react.useSuspense: false`
- Resource shape: `resources = { en: { translation: enJson } }`

### Baseline config pattern

```ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import en from "./locales/en.json";

const resources = {
  en: {
    translation: en,
  },
};

void i18n
  .use(initReactI18next)
  .init({
    resources,
    lng: "en",
    fallbackLng: "en",
    keySeparator: false,
    nsSeparator: false,
    returnNull: false,
    returnEmptyString: false,
    interpolation: { escapeValue: false },
    react: { useSuspense: false },
    debug: import.meta.env.DEV,
  });
```

## Boundary rules

- Translate in feature/route/form components.
- Keep primitive/base UI components translation-agnostic.
- Pass already translated strings into primitives.

## Key strategy

- Use hybrid keys and keep them stable and descriptive.
- Avoid dynamic key construction where possible.
- Prefer complete sentence templates with interpolation placeholders.

## Interpolation and pluralization

- Use interpolation placeholders for runtime values.
- Use explicit plural forms rather than string concatenation.

```json
{
  "day_one": "{{count}} day",
  "day_other": "{{count}} days",
  "Welcome, {{name}}": "Welcome, {{name}}"
}
```

```tsx
const label = t("day", { count: dayCount });
const welcome = t("Welcome, {{name}}", { name: userName });
```

## Conditional copy

- Use separate keys for materially different sentences.
- Keep branch selection logic in code and wording in translation files.

## Adoption reminder

- If i18n is not currently installed/configured in the target project, request approval before adding translation dependencies or new locale infrastructure.
- If project conventions conflict with this baseline, ask before deviating.
