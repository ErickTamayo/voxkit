# Internationalization

## Usage Pattern

```typescript
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();
  const userName = "John";

  return (
    <View>
      {/* Simple text */}
      <span>{t("Sign Out")}</span>

      {/* With interpolation */}
      <span>{t("Welcome, {{name}}", { name: userName })}</Typography>
    </View>
  );
}
```

## Translation Files

**Location**: `resources/js/i18n/locales/en.json`

```json
{
  "Welcome, {{name}}": "Welcome, {{name}}",
  "Sign Out": "Sign Out",
  "Sign In": "Sign In",
  "Email is required": "Email is required",
  "Invalid email address": "Invalid email address",
}
```

## Pluralization

Use `count` to enable i18next pluralization. Provide `_one` and `_other` keys:

```json
{
  "day_one": "{{count}} day",
  "day_other": "{{count}} days"
}
```

```typescript
const label = t("day", { count: 2 }); // "2 days"
```

## Conditional Copy

When a UI sentence can change based on data, prefer separate translation keys instead of building strings in code.

```typescript
const createdDate = format(job.created_at, "date");
const updatedDate = format(job.updated_at, "date");

const metaLabel =
  createdDate === updatedDate
    ? t("job.meta.created", { created: createdDate })
    : t("job.meta.createdUpdated", {
        created: createdDate,
        updated: updatedDate,
      });
```

## No Translations in Primitives

❌ **DON'T** add translation logic to base UI components:

```typescript
// ❌ Wrong - Button.tsx
export function Button({ label }: { label: string }) {
  const { t } = useTranslation(); // ❌ Don't do this in primitives
  return <Text>{t(label)}</Text>;
}
```

✅ **DO** translate in feature/implementation components:

```typescript
// ✅ Correct - SignInForm.tsx
export function SignInForm() {
  const { t } = useTranslation();
  return (
    <Button>
      <Button.Label>{t("Sign In")}</Button.Label>
    </Button>
  );
}
```
